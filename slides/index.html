07

Reduce — per-persona segmentation

Feedback.by_persona gives the coder persona-aware signal, not aggregate noise.

For each persona:

  • success rate & avg steps
  • distinctive actions — action types ≥ 2× cohort median
  • distinctive quotes — reasoning lines unique to this persona

"median user clicks 5×" is useless. "elderly_first_time abandons at step 4 with 'I don't understand why this isn't working'" is what the coder needs to write the right patch.

08

The signal in the persona's own voice

Stage 1 doesn't just tag failures — it surfaces what each persona actually said that no other persona said.

Per-persona quote extraction:

  • Take all reasoning lines from this persona's trajectories
  • Substring-fingerprint each against every other persona's reasoning
  • Keep the longest unique quote per trajectory; rank by length

Drops straight into a PR description, design review, or a coding-agent prompt. Persona voice replaces aggregate statistics.

Implementation grounding

Engine — 23 modules, ~6k LOC (engine only, excluding harness), pyright-clean on our code, Pydantic-validated everywhere.

  • schemas.py — Persona, Task, App, Action, Step, Trajectory, Feedback
  • map/worker.py — async per-cell loop with stuck-detection, retries, replay recording
  • io.py — streaming JSONL, fsync per step, crash-safe
  • harnesses/surfer/ — Surfer planner+grounder, vendored

Apps — JSONL app registry defines target URLs + tasks + success criteria; runtime k8s manifests deploy the apps themselves.

Both Pydantic-validated; teammate pushes registry adds, the engine consumes them as first-class data.

Live dashboard — Next.js + FastAPI, iframes Kernel's browser_live_view_url for real-time grid view (apps/dashboard/).

10

What's novel

  1. First CUA/BUA-based UserSim framework. Prior work is text-only chat eval (DeepEval, LangWatch, OASIS). We extend the user-simulator pattern from text → real browser, real DOM, real visual UI.
  2. Per-persona reduction. Feedback.by_persona surfaces signal traditional clustering hides; coder gets persona-specific fixes, not median-user platitudes.
  3. Held-out reward firewall. Two-stage reward gives a falsifiable measure of reward hacking in CUA-driven product loops — a domain we don't believe this exists for elsewhere.
  4. Provider-agnostic agent comparison. Same trajectory format across northstar, claude, surfer; agent comparison is one CLI flag.
  5. End-to-end reproducible. Apps + personas as Pydantic registries, JSONL trajectories, ffmpeg replay grids, deterministic Stage 1 — every claim re-runnable from git clone.
