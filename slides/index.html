05

Personas and feedback

105 personas across age, device, language fluency, tech literacy, prior experience. 5 hand-curated + 100 LLM-expanded with PersonaHub-style negative-example dedup, each with a generated avatar.

{"id":"elderly_first_time",
 "archetype":"68yo first-time online tax filer",
 "tech_literacy":"low","patience_steps":15,
 "quirks":["reads every word",
           "backtracks often",
           "worried about clicking wrong thing"]}

Explicit quirks injected into each rollout's system prompt drive measurably different trajectories — ~4× variance in step-count on identical tasks.

Per-persona feedback — not aggregate noise. Feedback.by_persona gives the coder:

  • success rate & avg steps per persona
  • distinctive actions — action types ≥ 2× cohort median
  • distinctive quotes — reasoning lines unique to this persona, substring-fingerprinted against all others

"elderly_first_time abandons at step 4 with 'I don't understand why this isn't working'" — that's what the coder needs to write the right patch, not "median user clicks 5×".

Drops straight into a PR description, design review, or coding-agent prompt. Persona voice replaces aggregate statistics.

06

Code changes and redeploys

The coder agent reads feedback.json, patches the target app's source, and redeploys — fully automated, isolated from production.

  • Clone the repo — each target app's source is cloned locally; the coding agent (Claude CLI) reads friction clusters + suggested fixes and proposes patches directly against the codebase.
  • Depot for fast rebuilds — Docker images are built via Depot for fast, cached remote builds. No waiting on local Docker.
  • Redeploy into live K8s — patched image is pushed and rolled out to a separate dev-* subdomain on our cluster, never touching the baseline deployment.
# coder/loop.py — one iteration
feedback = load("feedback.json")
patch = claude_cli.patch(feedback)

if patch.success:
    git commit -m "iter_N: auto-ux patch"
    depot build → push image
    kubectl rollout → dev-app.usersim.alexkreidler.com

# next MAP iteration targets the new deploy

The baseline stays untouched for A/B comparison. Iteration N+1 MAP runs target the patched deploy, closing the loop.

09

What's novel

  1. First CUA/BUA-based UserSim framework. Prior work is text-only chat eval (DeepEval, LangWatch, OASIS). We extend the user-simulator pattern from text → real browser, real DOM, real visual UI.
  2. Per-persona reduction. Feedback.by_persona surfaces signal traditional clustering hides; coder gets persona-specific fixes, not median-user platitudes.
  3. Held-out reward firewall. Two-stage reward gives a falsifiable measure of reward hacking in CUA-driven product loops — a domain we don't believe this exists for elsewhere.
  4. Provider-agnostic agent comparison. Same trajectory format across northstar, claude, surfer; agent comparison is one CLI flag.
  5. End-to-end reproducible. Apps + personas as Pydantic registries, JSONL trajectories, ffmpeg replay grids, deterministic Stage 1 — every claim re-runnable from git clone.
10

Future work

  • Benchmark between Tzafon / Holo / Claude. Same trajectory format across all three agents — run identical persona × task grids on each, compare success rates, step counts, friction patterns, and cost per trajectory. The AgentClient protocol makes this a one-flag swap.
  • Expand to longer-horizon tasks. Current tasks are single-session (login, create, browse). Real UX friction often spans multi-session workflows — onboarding funnels, multi-day retention loops, complex admin setups. Extend the worker to resume sessions and track state across runs.
  • Improve speed. End-to-end iteration time is currently bottlenecked by CUA latency (~5-10s per turn) and sequential grading. Parallelize Stage 1 grading, pipeline MAP and REDUCE overlap, and explore smaller/faster grounder models to cut per-iteration wall time.
